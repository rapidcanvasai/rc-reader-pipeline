# RapidCanvas Pipeline Reader - Sync Repository

This repository stores synchronized RapidCanvas workspace data organized by workspace and project.

## Repository Structure

```
rc-reader-pipeline/
├── [Workspace Name]/          # e.g., MiltonCAT
│   ├── [Project Name]/        # e.g., Sales Analytics
│   │   ├── recipes/           # Python transformation recipes
│   │   │   ├── recipe1.py
│   │   │   ├── recipe2.py
│   │   │   └── ...
│   │   ├── datasets/          # CSV datasets (exported from entities)
│   │   │   ├── dataset1.csv
│   │   │   ├── dataset2.csv
│   │   │   └── ...
│   │   └── pipeline.json      # Complete canvas metadata
│   └── [Another Project]/
│       ├── recipes/
│       ├── datasets/
│       └── pipeline.json
└── [Another Workspace]/
    └── ...
```

## About

This repository is automatically synchronized from the RapidCanvas Pipeline Reader application. Each workspace and project maintains its own folder structure for:

- **Recipes**: Python code for data transformations and ML pipelines
- **Datasets**: Exported CSV data from RapidCanvas entities
- **Pipeline Metadata**: JSON files containing canvas structure, nodes, and edges

## Synchronization

Files in this repository are synced using the GitHub Integration feature in the RapidCanvas Pipeline Reader application. Each sync creates a commit with the latest state of the workspace.

## Usage

You can:
- Browse recipes and datasets for any workspace/project
- Track changes over time through commit history
- Use the code and data for analysis or deployment
- Integrate with CI/CD pipelines
